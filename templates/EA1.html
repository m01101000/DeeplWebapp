Results

In deep learning, particularly in image classification tasks, neural networks are trained to recognize patterns and features within images. These networks learn from large datasets, adjusting their internal parameters (weights and biases) during a process called training. The goal is for the network to learn representations of the input data that allow it to make accurate predictions about new, unseen data.

In this case the results show that the classification strongly depends on the perspective and context of the respective image. This becomes clear when looking at the third correctly classified image and the third incorrectly classified image. The third correctly classified image shows a boat moving on water, the confidence is given as 94%. If you look at the individual features such as colors and shapes, you will notice that there is not much to see in this picture other than the boat. Only the dark blue water and the white spray of the boat can be seen. If you look at the misclassified image of the boat, which was classified as a missile with a 33% confidence, you can see that there are very different colors and shapes in addition to the boat. The picture shows black wheels of a turquoise vehicle. This vehicle is standing on a gray area and is loaded with the boat. This completely different representation makes pattern recognition and thus also neural assignment more difficult.

In the context of the described scenario, let's assume the neural network has been trained on a diverse dataset that includes images of both boats and missiles, as well as other objects. During training, the network learns to associate certain features and patterns with each class. For example, it might learn that boats often appear surrounded by blue pixels (representing water), have elongated shapes, and exhibit certain textures like white spray. Similarly, it might learn that missiles tend to have cylindrical shapes.

Another example is the first misclassified image of a smartwatch. The picture shows a completely white background and the clock shown only has the three hands for hours, minutes and seconds as the only traditional features of a clock. It is also difficult to determine the shape of the case and band because they are the same color. In my opinion, the neural network may not recognize the smartwatch because there are no contextual features beyond the three traditional watch hands.

In essence, the differences in classification between these images highlight the importance of context and perspective in image classification tasks. The neural network's ability to correctly classify an image depends heavily on the similarity between the features it has learned during training and the features present in the input image. When the input image deviates significantly from the learned features or context, as in the case of the misclassified boat, the network may struggle to make accurate predictions.

Technical approach:

Light/Dark Mode Implementation:



This code detects the user's preferred color scheme using window.matchMedia, specifically checking if the user prefers a dark color scheme. Based on the result, it sets the colorScheme variable to either 'dark' or 'light'. Then, it dynamically updates the href attribute of the tags with IDs site-css and ea1-css to load the appropriate CSS files based on the detected color scheme.


Deviation from task (no classify button):

When creating the form structure, I incorporated a deviation from the task. There is no classify button. Images are automatically classified after uploading.

For classification, this web application first checks whether the size of the uploaded file is correct (I set it to 400x400 pixels), then classifies the file and then the results are classified and saved or discarded by the user.

Image size check


This JavaScript function handleFiles is responsible for checking the dimensions of the uploaded image. It takes a list of files as input, assumes the first file is selected, and checks if its dimensions are 400x400 pixels. If the dimensions are correct, it proceeds to upload the image. Otherwise, it displays an alert prompting the user to upload an image with the required dimensions.

Automated image classification


The setup function initializes the canvas and triggers the classification process. It creates an HTML image element from the uploaded image source and appends it to the HTML 'img' div. Meanwhile, the gotResult function handles the classification result by extracting the label and confidence score. It updates hidden input fields with this information and dynamically creates HTML elements to display the classification result.

Workflow


This HTML form represents the workflow for evaluating and saving the classification result. It includes buttons for users to indicate whether the classification is correct or incorrect. JavaScript functions manage the selection of these buttons and ensure that an evaluation is chosen before proceeding with form submission. Upon submission, the classification result can be saved.
